Philosophy

Bauhaus is an administration package with expressive, elegant syntax for the Laravel framework. We believe backend development should be easier, faster, more flexible and tightly integrated with the provided framework (Laravel in this case). Bauhaus attempts to take the pain out of web-development by providing tools and functionality to easily integrate a fast, clean and easy backend system into any new or existing website.

We share the same philosophy as Laravel itself, meaning that "we aim to make the development process a pleasing one for the developer without sacrificing application functionality. Happy developers make the best code". Therefor, we've looked, tried, and tested other backend systems from a wide variety of languages and frameworks and combined the very best ideas to create Bauhaus.

Installation

To install Bauhaus in your latest and greatest Laravel project, use composer and simply add krafthaus/bauhaus to your composer.json file:

"require": {
	"laravel/framework": "4.2.*",
	"krafthaus/bauhaus": "dev-master"
}

Then use composer to install the package.

$ composer update

Registering

Once installed, you have to register the service provider in app/config/app.php the providers array:

'providers' => array(
	'KraftHaus\Bauhaus\BauhausServiceProvider'
)

Configuration

Once everything is installed and registered, you need to tell composer where to find the bauhaus files. Add the admin folder to the app/ directory and put the following line in your composer.json file: 

"autoload": {
	"classmap": [
		"app/admin"
	]
}

For this to take effect, run composer dump-autoload.

Then publish the config file with php artisan config:publish krafthaus/bauhaus. This will add the main bauhaus config file in your application config directory.

And last but not least you need to publish to package's assets with the php artisan asset:publish krafthaus/bauhaus command.

Getting Started

Lets say you're building a blog (of course this is only limited to your imagination) an need to have some way to manage, view and update your blog posts. Bauhaus will help you create exactly just that in a couple easy and quick steps.

Below is explained what those steps are.

  • We will need to create a model (Eloquent model)
  • A migration to create our database table
  • And a Bauhaus controller to manage our posts
Bauhaus tries to streamline this process by providing one command to rule them all: 
$ php artisan bauhaus:scaffold --model=post

Running this command will create a couple of files:

  • A Bauhaus controller in the app/admin directory
  • A migration in app/database/migrations
  • A new model in app/models

Open the newly created migration at app/database/migrations/{timestamp}_create_posts_table.php and change it into something like this:

class CreatePostsTable extends Migration
{
	public function up()
	{
		Schema::create('posts', function(Blueprint $table) {
			$table->increments('id');
			$table->string('title');
			$table->text('content');
			$table->timestamps();
		});
	}

	public function down()
	{
		Schema::drop('posts');
	}
}

Now open app/admin/PostAdmin.php in change the boilerplate code to the following:

class PostAdmin extends KraftHaus\Bauhaus\Admin
{
	public function configureList($mapper)
	{
		// Render the title in the overview
		$mapper->identifier('title');
	}

	public function configureForm($mapper)
	{
		// Render a text field and a wysiwyg editor
		$mapper->text('title');
		$mapper->wysiwyg('content');
	}
}

Now visit http://your-project-url/admin/model/post and voila, you have a working backend where you can view, edit and create your posts.
