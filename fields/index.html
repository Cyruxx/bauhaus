Fields

A detailed documentation about every field type in Bauhaus.

Boolean

The boolean field type renders a checkbox.

NOTE: The boolean field type should be represented as an integer field in your database (usually TINYINT(1)).

Description

$mapper->boolean( string $field )

Parameters

$field
The field name in your model.

File

This field renders an file input field and seamlessly handles file uploads.

Description

$mapper->file( string $field )
	->location( string $location )
	->naming( string $naming [ original | random ] ) // optional

NOTE: The file field requires the use of the location( string $location ) method.

Parameters

$field
The field name in your model.
$location
The location to upload your file to.
$naming
Whether or not to randomize the filename.

Example

// Upload the great_document file to the uploads/documents directory and saving it with a random name
$mapper->file('great_document')
	->location('uploads/documents')
	->naming('random');

Number

Renders a number formatted version of the TextField.

Description

$mapper->number( string $field )
	->decimals( int $decimals = 2 ) // optional
	->separators( array $separators = ['.', ','] ) // optional

Parameters

$field
The field name in your model.
$decimals
Sets the number of decimal points.
$separators
Sets the decimal point and the thousands separator.

Select

Renders a dropdown (select field) based on the given options and value.

Description

$mapper->select( string $field )
	->options( array $options )

Parameters

$field
The field name in your model.
$options
An array of options (associative and non-associative).

Examples

// as a non-associative array
$mapper->select('gender')
	->options(['Male', 'Female']);
// as an associative array
$mapper->select('gender')
	->options([
		'm' => 'Male',
		'f' => 'Female'
	]);

Belongs To Relationship field

Renders a dropdown (select) list with related items based on the relationship set on your model.

NOTE: This field requires the use of the display( string $field ) method to know which field to display.

Description

$mapper->belongsTo( string $relationship )
	->display( string $field )

Parameters

$relationship
The name of the belongsTo method on your model.
$field
The field name on the other side of the relation.

Examples

// Your model file
class User extends Eloquent
{
	// Setup the relationship with the Posts model
	public function posts()
	{
		return $this->belongsTo('Posts');
	}
}

// Your Bauhaus file
class User extends Admin
{
	public function configureList($mapper)
	{
		// This renders the post title as a string in your list view.
		$mapper->belongsTo('posts')->display('title');
	}

	public function configureForm($mapper)
	{
		// This renders a list of posts in your form.
		$mapper->belongsTo('posts')->display('title');
	}
}

Belongs To Many Relationship field

Renders a multi-dropdown (multi-select) list with related items based on the relationship set on your model.

NOTE: This field requires the use of the display( string $field ) method to know which field to display.

Description

$mapper->belongsToMany( string $relationship )
	->display( string $field )

Parameters

$relationship
The name of the belongsToMany method on your model.
$field
The field name on the other side of the relation.

Examples

$mapper->belongsToMany('users')
	->display('email');
