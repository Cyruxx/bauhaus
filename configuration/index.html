Configuration

How to customize Bauhaus to your needs.

The basics

Accessing the Bauhaus interface

If you've followed the steps in the installation section the (empty) Bauhaus interface should now be accessible via http://yoursite.local/admin.

Bauhaus helps you manage your data using a graphic interface that will let you create, edit, update, delete your model's instances. Those actions need to be configured, which is done using a Bauhaus controller.

A Bauhaus controller represents the mapping of your model to each action. In it, you decide which fields to show on a listing, which to use as filters or what to show in a form.

A basic controller

The easiest way to create a Bauhaus controller for your model is to extend the KraftHaus\Bauhaus\Admin class.

Suppose your website has a Post model. This is how a basic Bauhaus controller should look like:

class PostAdmin extends KraftHaus\Bauhaus\Admin
{
	// Fields to be shown on the index view
	public function configureList($mapper)
	{
		$mapper->identifier('title');
		$mapper->string('slug');
		$mapper->belongsTo('author')->display('fullname');
	}

	// Fields to be shown on the form view
	public function configureForm($mapper)
	{
		$mapper->text('title')->label('Post title');
		$mapper->belongsTo('author')->display('fullname');
		$mapper->wysiwyg('body');
	}
}

Implementing these two functions is the first step to creating a Bauhaus controller. Other options are available that will let you further customize the way your model is shown and handled. Those will be covered later on in this manual.

Now that you have created your first Bauhaus controller, you can preview it at http://yoursite.local/admin/model/Post. (Keep in mind that the part after model in the url is the class name of the BauhausController without `Admin`).

Architecture

The architecture of Bauhaus is heavily inspired by the Sonata Admin Bundle and the Django Admin Project, which are truly great projects.

If you followed the instructions above, you should by now have a working Bauhaus controller.

The Bauhaus controller maps a specifc model to the rich CRUD interface provided by Bauhaus. In other words, using your controller, you can configure what is shown by Bauhaus in each CRUD action for the associated model. By now, you've seen two of those actions in the paragraphs above: list and form. However, a fully configured Bauhaus controller can define more actions:

  • list: The fields displayed in the list table.
  • filter: The fields available for filtering the list.
  • form: The fields used to create/edit the model.
  • scope: The query scopes for filtering the list.
